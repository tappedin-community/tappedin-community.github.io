Tapped In 2: Course Manager

Functional Specification

Version 3.0

 

1. Overview

 

      This document describes the functional specifications for the Course Manager for Tapped In Version 2.0.  The core implementation of the course management system is licensed from Prometheus, a third party.  The Prometheus source code will have to be modified and appended to accommodate 1) the features not already provided by Prometheus, 2) the proposed interface with Tapped In, and in later versions, possibly 3) integration with other course management systems such as Blackboard and Yahoo!.  An open dialogue with the writers of Prometheus will maintained such that whenever possible, large changes and additions to the Prometheus source code base can be implemented by Prometheus rather than the Tapped In team.

      The Course Manager is completely integrated into the Tapped In community and interface.  The course management functionality primarily appears in the Tapped In content area (see Integration into Tapped In below) when a user is in the Tapped In room for that course.  This is referred to as the “course room” throughout the document.  Only Tapped In members that are enrolled as teachers, students, etc. in the course can access the course room.  The processes of finding, enrolling in and creating courses is specified in this document.

      After laying out a few assumptions, the specification begins with descriptions of users, course attributes, an overview of course features, an explanation of how users find and enroll in courses, then walks through the proposed interface organization for a course room, describing where and how users would access various functions and execute tasks.

 

2. Assumptions

 

This specification is written with several assumptions in mind:

 

      Tapped In has a number of features that will be applicable to the course manager.  These include but may not be necessarily limited to a chat facility, private messaging, and a means to share files and links in Tapped In.  The course manager specifications for some portions may allude to designs for other Tapped In features and be less specific than for functionality only described here.

      There is little distinction in the specification between functionality and interfaces of the course manager that are provided by Prometheus and those not provided by Prometheus.  The Prometheus code will be leveraged to meet the requirements of the specification as much as possible, with the remaining functionality to be either implemented by Prometheus, independently implemented by modifying Prometheus source code or pre-written open source code modules (e.g. a chat facility) to be integrated into Tapped In.  In all likelihood, several features provided by Prometheus will be unused.  It is worth noting that as more modification and in-house development is required, the implementation process will lengthen.

 

3. Users

 

      There are primarily two types of users of a course management system: teachers and students.  Accordingly, there are two basic interfaces (teacher and student) which are very similar for the most part.  The most pronounced difference between the two interfaces is the ability of the user viewing the teacher interface to edit content, use the gradebook and access other teacher-specific utilities.  It is important to keep in mind that the user roles described here are in context with the course in question, and do not serve as Tapped In user roles at large.  For example, a teacher in one course might be a student in another.

      Prometheus provides four levels of course level access control (see excerpt below).  Not unexpectedly, a user in a teacher role would see the teacher interface and one in a student role would see the student interface.  The teaching assistant level would view a version of the teacher interface with unusable functions not appearing in the interface and similarly, the shadow level would view a version of the student interface with unusable functions not appearing.  Tapped In is room based, and each room has a single owner, which in this case will be an instructor level user.  This instructor is the primary instructor.  Even if a fellow instructor or teaching assistant level user has full edit access permissions (as described in Settings below) only the primary instructor can delete the course.  All instructors always have full edit access.

4. Course Attributes

 

4.1. Tenant Organization

 

      As of now, all Tapped In courses will be sponsored by a tenant organization (e.g. Pepperdine, NCREL, etc.).  So only a tenant organization member can create a course, and once created, that course would be associated with that tenant.  In the future, course management may become available to non-tenant members and therefore there may be non-tenant courses.  Each course may be associated with only one tenant.

 

4.2. Privacy Settings

 

      A course is specified as either public, private, or public to tenant members.  Designating a course as public means that any Tapped In member can apply to enroll and that the course appears in the tenant course directory and the Tapped In course directory.  A private course does not appear in any directories whatsoever and users can enroll by invitation only.  Finally, a public to tenants course appears in the tenant and Tapped In directories only for members of the tenant organization with which the course is associated.  For example an NCREL public to tenants course would appear in the directories of an NCREL Tapped In user but not those of a Pepperdine user.  Likewise, only members of the appropriate tenant organization can apply to enroll to a public to tenants course.  In short, a public to tenants course looks public to a tenant member and private to a non-tenant member.

      If a course does not appear in a user’s directories, then that course never appears in the search results for that user in any case (enrolled or not, any privacy setting).  Any member can be invited by an instructor to join a course, regardless of privacy settings.  None of the three settings allows users to enroll in courses without instructor approval (i.e. even public courses enrollment applications require instructor approval although not an invitation).  The instructor specifies the privacy setting upon course creation.  An instructor can make a course more public (e.g. from private to public to tenants, or public to tenants to public), but not more private (e.g. not from public to private).

      Once a user is enrolled, that course appears in the Me course directory, the Tapped In directory, and if applicable (i.e. the user is a member of the course tenant organization), the tenant directory, regardless of the privacy setting of the course.  Similarly, an appropriate search will always return courses that the user is currently enrolled in, regardless of the privacy setting of the course.

4.3. Course Dates

 

      When a course is created, the course dates are specified.  This consists of a starting and ending date for the course. (Should they be able to be changed?)

 

4.4. Course ID & Password

 

      When a course is created the instructor chooses a password for the course and the course is assigned an ID number.  The instructor can modify the password once the course is created.  (How are these used?)

 

4.5. Keywords

 

      Every course can optionally be associated with one or more established Tapped In keywords.  The instructor can specify these when creating the course and modify them once the course is created.  These keywords are used in the Tapped In search function.  For example, if “Math” is checked as a keyword for a public course, then the course will appear in the search results of a user who searches for courses and groups associated with “Math”.

 

5. Outside the Course Room

 

      Course management functionality is encapsulated in a course room, but Tapped In users must have a way to access and create course rooms. The primary methods of doing this are by using the directories at the top of the Tapped In interface.  These are described below after which scenarios describe the processes of creating a course, enrolling students, and teaching a course (teacher/teaching assistant) as well as choosing, enrolling in, and taking a course (student).  Activities and tasks performed inside the course room are described in the Course Room Interface Organization section.

 

5.1. Course Sections of Directories & Search Results

 

      At the top of the Tapped In interface are tabs that are called directories, two of which, Me and Tapped In, always appear.  Any additional directory tabs are for tenant organizations.  Each directory has Course section.  The Me directory lists all the courses in which the user is enrolled.  The tenant directory or directories lists all the courses offered by the tenant organization that are public to the user.  The Tapped In directory lists all courses public to the user offered by all tenants.  A user can also conduct a course search from the search area of Tapped In which results in a course listing.  If the user is a tenant member, there is also a “Create A Course” link on any of the directory course listings.

      Various listings are available for the different directories, including alphabetical, by tenant, and by category.  Indications (by symbol, color, font style, etc.) of the privacy setting and if the user is enrolled is present in some listings.  Clicking on an entry in a listing or a search result gives you an information screen about the course if you are not enrolled and takes you to room if you are enrolled.  If no courses are available for a given tenant or overall, then a message should be printed in the directory to that effect.  (Print the number of unavailable (private) courses?) For example, a course listed in the Tapped In directory alphabetically might look like this:

 

      <door> <private> CS 101 (Pepperdine: zazh)

 

where <door> is a door symbol indicating that the course is a room so the user is enrolled, <private> is some symbol indicating the course is private (to this user), Pepperdine is the tenant, and zazh is the owner of the course, i.e. the primary instructor.

5.2 Course Information Page

 

      The course information page is not part of the course room itself, but rather information provided by the course instructor(s) about the course to non-members.  This includes, at the least, a block of text written by the instructor, the tenant sponsoring the course, the number of students, the names of the instructors and teaching assistants, the keywords the course matches, and the course dates.  Somewhere on this page should be an “Apply to Enroll” to allow the user to apply to enroll in this course.

 

5.3. Tasks Outside the Course Room

 

5.3.1. Finding a Course (Not Enrolled) - All Roles

 

      There are several ways for a user to find an existing course they are not enrolled in.  The two guaranteed ways are to either search for the course using the search function or to look in the directories at the top of the Tapped In interface.  However, courses will only appear in either of these places if the course is public to the user (see Privacy Settings).

      Other places a user might find a course is to try and take a passageway from another room to that course room.  Presumably, tenant organizations will, as a group, have a group room which would probably have a passageway or a link to tenant courses.

      Any of these options would result in getting the information screen for that course because the user is not yet enrolled.

 

5.3.2. Finding a Course (Enrolled) - All Roles

 

      A user that is enrolled can get to a course room by using any of the methods that are listed for finding a course in which the user is not enrolled.  In addition, however, the course appears in the Me directory course section.  Once enrolled, a user can also designate the course room as a favorite place or build a passageway from the user’s home office to the course room for easy access.

 

5.3.3. Enrolling in a Course (All Roles but Primary Instructor)

 

      To enroll in a course, the user must first find the course (see Finding_a Course (Not_Enrolled) – All Roles), and, having arrived at the information page for that course, click on the link that says “Apply to Enroll”.  This will send a message to the primary instructor of the class of the application for enrollment.  The primary instructor will then either approve or reject the user’s enrollment and the user will be notified.  After that, the course will appear in the Me directory and clicking on the course name will take the user to the course room rather than the course information page.

 

5.3.4. Creating a Course (Primary Instructor)

 

      A tenant member can create a course my going to the Courses portion of any of the three directories and clicking on “Create a Course”.

 

6. Course Features

 

      The Tapped In Course Manager essentially provides mostly the same features as the Prometheus course management system but in the broader online setting of the Tapped In community at large.  Tapped In users can chat (with other users in the same room and taking advantage of chat support tools) and private message or e-mail any Tapped In user.  Most Tapped In rooms (including course rooms) have a discussion board where users can post messages in threaded discussions.  Passageways between rooms allow users to move more easily between rooms.  Group and course rooms allow file uploading and downloading into a file space for that room.  Many of these features are partially or fully present in Prometheus but will be re-implemented or adapted to Tapped In as a whole.

      The features remaining specific to course rooms are the syllabus, which provides all kinds of course information, providing course materials online, allowing for course work to be graded and submitted online, and course groups.  The Tapped In version of a course group will simply be an instance of Tapped In group with the addition of an explicit association of the group with a course.  See Settings for more information on course groups.  The details of course specific features can be found in the Course Room Interface Organization section.

 

7. Course Room Interface Organization

 

7.1. Prometheus

 

      Prometheus uses a relatively flexible edit form paradigm, which allows teachers to create and modify content.  Most fields can be renamed and fields left empty do not appear in the non-edit view. This basic paradigm is upheld in the interface that follows, although whereas Prometheus essentially allows the user to edit a very long page, listing all the fields, the interface specified here edits small portions of information in place.  That is, on a page featuring three main blocks of information, the user can edit the second while the first and third still appear static.  The Prometheus course manager is divided into distinct modules.  Any module of the course manager can be disabled for any particular user in a course.  This level of flexibility is probably unnecessary and if disabling modules is allowed at all, it will be for all users in the course.

      Prometheus also provides tools for editing field content, including 1) importing a file, 2) equation creation and insertion, 3) creating an internal link to another course manager unit (e.g. a day in the calendar) and 4) spell checking.  In Prometheus, all four tools are provided for every single text box.  Some alternatives to this approach are to provide the tools in some central location for use at any time or to pick and choose which tools are to be available for particular fields.

      Another detail worth noting is that Prometheus field contents are printed in the html as to allow html tags to be executed, which while convenient, could be a little problematic in regards to security.  It would be worthwhile to investigate how and to what degree Prometheus parses field contents.

 

7.2. Integration into the Tapped In Interface

 

      The current concept of a Tapped In room currently consists of the content area within which is a sub-navigation bar for the room, and a detachable chat/private message window.  The sub-navigation bar is on the left side of the screen and has a picture of the room and a set of sub-navigational links.  A course room is a Tapped In room with added course managing functionality, i.e. with additional or slightly different sub-navigational links allowing the user access to course features. As mentioned above, course manager functionality is only active when the user is in the room for particular course (all courses must have a room).

      Some features that are mentioned in this document as part of the course manager would appear throughout Tapped In (chat, passageways, for example) while others are specific to course management.  Functionality specific to course management will always appear as links in the sub-navigation bar on the left side of the content area, rather than the interface at large. 

      The sub-navigation bar organizes the links in hierarchy.  Top level links are always visible.  Second level links are visible only for the top level page the user is currently viewing.  Any page that is three levels or deeper into the hierarchy of the page will never appear in the sub-navigation.  When the user is viewing a particular page, the link for that page in the sub-navigational bar will be disabled.

There are also going to be three blank stickies for users with room configuration edit access to configure as desired.  The editing abilities for items on this screen are also accessible in Settings: Room Configuration.

7.5.2. Student

 

      The first two sections of Syllabus provide access to information about the course that is not associated with a particular date.  It is divided into two pages: Course Description & Staff, and Policies. Course Description lists the course title and number, meeting time/locations, and the course description. Staff provides information about the instructor(s) of the class with name and contact information, etc. Clicking an instructor’s e-mail address takes the user to the e-mail pop-up window.  The Policies section describes any course policies the instructor wishes to include.  These might be Prerequisites or Method of Evaluation, for example.

      In the Calendar section, the student can view an integrated calendar displaying all events for the course including lecture topics, assignments, quizzes and tests, reading assignments, and other course-related events.  Students can merge one or many events into their personal calendar.  Clicking on course work events (assignments, quizzes, and tests) should take the student to that entry on the Course Work page.  Similarly, reading assignments should take the student to the Course Materials page.

      Ideally, this calendar will be very much like a typical Tapped In calendar but with added functionality for courses.  As such, much of this section may change once the functional specification for the Tapped In calendar has been written.

 

7.5.3. Teacher

 

      For course-related content, each item listed as an editable unit above has an Edit button next to it which allows content to be edited and saved in context.  Teachers other than the primary instructor (i.e. the owner of the room) would be able to edit this tab only if they have edit privileges for this tab.  An unlimited number of meeting times, instructors, optional instructor info fields, and course policy topics may be added.

      Teachers should be able to set a range of dates to display for the course, add new events, edit current events, including changing the date of events.  For events that appear that are detailed in another part of the Course Manager, assignments for example, the teacher can click on these items to edit them in that section.  The dates specified there will automatically appear on the calendar here.

7.6.2. Student

 

      In this section, the student can find information about any materials needed for the class.  These may include readings, handouts and online resources.  Readings are subdivided further into reading groups, the default being Required and Recommended.  Handouts are documents available for downloading and/or online viewing.  Online resources are any web links the instructors would like to post for the course.

      Any downloadable documents and links will be duplicated in the Files & Links tab.

 

7.6.3. Teacher

 

      Each of the three sections (Readings, Handouts, Online Resources) course materials can be viewed and edited by the teacher.  Each section itself can be renamed and are fairly flexible.  A “reading” can  is simply a description of any sort of course material.  The names of current reading groups can be edited, and readings can be moved from one group to another.  New reading groups and new readings can be added.

      Handouts can be online documents, available for online viewing and downloading, or binary files only available for download.  They can be on the web at large or in the file space for this course.  The teacher can choose from existing files in the course file space.  Files can be uploaded into the course file space in Utilities.  New handouts and online resources can be added.

 

7.6.4. Other Possibilities

 

      You might want to allow the teacher to re-order the readings, perhaps putting little up/down buttons next to each reading.  Otherwise a teacher would have to painstakingly cut and paste to rearrange a long list of readings.  Prometheus provides comments for online resources (which it refers to as web links).  Right now, that isn’t in the specification.

7.9.3. Teacher

 

      The settings page allows instructors and teaching assistants with settings edit privileges to modify the settings for the course.  These include the configuration of the room itself, configuration of course attributes, management of student course groups, and management of those enrolled in the class.

      The room configuration page allows the user to configure attributes common to all Tapped In rooms, such as the name of the room, the appearance of the welcome page, etc. 

      The course configuration room allows the user to change course settings, such as the privacy setting, search keywords, or the password for the course. 

      At the group management page, the user can create, edit, and delete student groups.  A student group is a normal Tapped In group with a simplified creation process and in which the instructor of the course has some owner-like privileges even if he or she is not the owner.  The initially chooses only the name and description of the group and can populate and remove students from the groups.  Student groups have the following features by default:

-         transcripts are on

-         group is private

-         tenant is the same as course tenant

-         group expires on course end date

-         room picture uses course room picture

In addition, a two passageway is automatically created between the course room and the group room.  The instructor can choose to either make him/herself the owner of the group or not a member of the group at all.  However, even if the instructor is not a member, they still have the ability to remove or add group members and can delete the group.  Course groups are explicitly linked to the course.

      Finally, the student management page provides a means to invite students, teaching assistants, and instructors to enroll in the course, drop any participant in the course, and configure edit privileges for teaching assistants.

      Students and shadow users never have access to any settings pages, instructors always do, and teaching assistant accounts can be allowed settings access or not.

Description from Prometheus:

 

Course Building and Management

 

·         Course Content Creation. Prometheus organizes course content around a syllabus model. Using a self-guided, question-and-answer format, instructors move easily through content creation. Guides within Prometheus are mere suggestions. Instructors can place content in suggested fields or disregard them. Any disregarded fields will not be visible to students in the syllabus. Because the content creation model in the syllabus section is consistent throughout Prometheus, instructors who have reached a comfort level there can easily create content in any section. The Prometheus course creation model works like this. The instructor

·         titles the content;

·         enters new content or imports existing content;

·         uploads text, multimedia, PowerPoint, or other files to enhance content;

·         associates notes with content files.

 

·         Modular Organization, Prometheus provides instructors with the ability to create learning objects outside the course context, group them into "content modules," then insert and rearrange them within courses. Such modules might contain outline items, assignments, tests, projects, lectures, files, and discussion topics.

 

·         Modular Content Distribution, Modular learning objects make content independent of any particular course, allowing reuse in multiple courses. Changes made to a module replicate across courses, enabling centralized content management. Instructors can reassign which modules are associated with which courses using a simple drag-and-drop interface.

 

·         The Prometheus summer 2001 release will allow distribution of entire course "cartridges.

 

Testing and Grade Book

 

  • Prometheus allows for quantitative assessments using multiple-choice or other objective testing formats as well as open content assessment, such as short answer and essay formats.

 

  • The testing feature allows online administration of tests and surveys. From the testing section, instructors can create and edit tests and surveys, grade tests, and view results.

 

  • Prometheus can be set to post online tests and corresponding grades automatically to a grade book. The grade book lists all students registered for a course. Scores edited in the testing section will automatically be updated in the grade book. Instructors can grant students permission to see only their own grades, anonymous grades for the entire class, or the grade book can be closed to student view entirely.

 

  • Instructors can post grades by student or by item. They can also add comments, reset, edit, or delete the entire grade book or specific items within the grade book. Instructors can email a student by simply clicking his or her name in the grade book.

 

  • Instructors can also download their grade book to Microsoft Excel by clicking one button.

Personal Assessment Tool2

Provide scaffolding for users to assess their teaching skills and identify the skills that they want to improve. This self assessment is based on Self Assessment Standards for Professional Development.

 

This self assessment will be optional. Users can choose whether or not to take the survey. Results will be shared with institutions anonymously.

 

Users can take the self-survey at any time. Results are logged with dates and shown on a grid so that users can see their progress.

 

The results can be printed or emailed to be shared with others.

 

The format of the survey is one question with optional answers per page. The number of questions left to be answered appears on the page, indicating where the user is in the process. There are no “right” or “wrong” answers.

 

There are levels to the self-assessment. A user may be guided to a different level given their previous results. More TK on this from Barry Kauffman. There may also be standards for PD providers.
