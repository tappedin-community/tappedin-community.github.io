Tapped In 2: “Tapped In” and “Tenant” Directories

Functional Specification

Version 4.0

 

1. Overview

 

At the top of the Tapped In interface are two to four directory tabs.  The two always present are “Me” and “Tapped In”.(1)  The third (2) and fourth (3) directory tabs are for Tapped In tenants.  Users may belong to as many tenants as they want, but only two tenant tabs are displayed in the interface.  This specification describes the tenant and Tapped In directory tabs, which are quite similar.  The Me tab is described in another document.  Throughout this document, tenant and Tapped In will be described as meta-groups to distinguish them from normal Tapped In groups.

These directory tabs allow the user to easily access a directory that lists the members, courses, groups, etc. for the meta-group.  Once the tab is clicked an overlay (1) appears over the interface to indicate that the information presented is not associated with whatever room the user happens to be in.  A horizontal row of sub-tabs (1) appear to allow the user to navigate in the directory.  Breadcrumbs will appear below the sub-tabs to assist in any navigation inside the sub-tab. (2)  See the example image after the Sub-Tabs section, which shows the CNBRC tenant directory tab open to the People sub-tab.  The breadcrumbs indicate that the user is looking at the Mentors page of the People sub-tab.

2. Sub-Tabs

 

Tenant and Tapped In directories have the same set of sub-tabs.  These are:

 

  • Welcome: An introduction to how to use this directory for Tapped In or the tenant.
  • People: A list of the members of the meta-group.
  • Places: A list of rooms owned by or associated with the meta-group, including course rooms, group rooms, and offices of members
  • Courses: A list of public courses offered by the meta-group. See the Course Manager specification for details.
  • Groups: A list of public groups owned by members of the meta-group. See the Groups specification for details.
  • Calendar: A Tapped-In calendar for the activities of the meta-group. See the Calendar specification for details.

 

Priority Summary: See other specifications or details below.

2.1. Welcome

 

The welcome sub-tab is a static page (1) or set of static pages (3) that provide the user with brief information about the meta-group and instructions on how to use the directory.  This might include a basic history of the meta-group, the purpose of the meta-group, a link to the primary room for the meta-group, some statistics, etc.  The meta-group in question should have some control over the content of this page. (2)  The purpose of this tab is to give users a starting point in using the directory for the meta-group in question.  For a tenant in particular, more information will be available in the group room for that tenant.

If this sub-tab includes more than one static page, the sub-navigation (3) will be extremely simple, either a sequential next/back ordering or sub-navigation links across the top.  These pages should avoid having a sub-navigation scheme on the left that is too similar to those of group and course rooms to avoid confusion.

2.2. People

 

The people sub-tab is a list of the members (1) of the meta-group.  This list is presented alphabetically showing the member’s real name followed by his or her Tapped In username, location in Tapped In (with a door icon), (1) mood, idle time (2), and sometimes tenant organization. (2)  In future versions, there may be an option to view the list alphabetically by username (3) or location. (2)  Next to each member that is currently logged on is a face icon.  Clicking on this icon puts that member in the private message dock of the user. Clicking on the username of the member allows you to view their profile. (1)  Clicking on the location of the member takes the user to that location in Tapped In. (2)  If the room in question is unavailable for some reason, the door icon shows a closed door and clicking the location takes the user to the information page for that room.  If a user is offline the location is listed as “Offline” (unclickable).  If a member is unavailable, the username appears grey as opposed to black. (1) Offline members are not initially shown the user can click on a link that makes them appear. (1)

If the tab in question is the Tapped In directory tab, there is also an option to view the list by tenant. (3)  If a member belongs to more than one tenant, they appear under all tenants in which they are a member.  If a Tapped In user is a member of any tenant organizations, then they appear in parenthesis in the alphabetical Tapped In people listing.  Some examples of what listings may look like follow below (where Pepperdine and NCREL are example tenants).

Once the member directory gets beyond some threshold size (e.g. 100 results), it may need to be “chunked”.  Instead of loading the entire member directory, the first 100 results would appear, along with individual links to the next 9 chunks and a final link to the next 10 chunks.  This is very similar to common methods of grouping search results.  Once a user is into the second set of 10 chunks a link to the previous 10 chunks would appear.  Each chunk is named according to who the list is sorted.  If the listing is sorted by username the first chunk might be “a-be”, by name, the second chunk might be “Bedf-Cha”.  A list sorted by tenant or location may make each chunk an actual room or tenant.

2.3. Places

 

The places sub-tab is a list of all the places (1), i.e. rooms, associated with the meta-group.  These fall into four categories: meta-group rooms, course rooms, group rooms, and personal offices.  The list is always organized by these categories, and then alphabetically by the room name.  If the meta-group is Tapped In, the list can be organized by tenant (3), and then by category, then alphabetized.

The room name is listed first followed by the owner (1).  If the listing is for the Tapped In tab and it is not organized by tenant, then the tenant is also indicated (2).  There are also the following clickable icons next to each room:

 Clicking the name of the room is equated with clicking the door icon.  If the room is unavailable, the door icon appears closed, and clicking the door or the name of the room takes the user to the information page for that room.  The information page can always be viewed by clicking the “i” icon.  Course and group rooms have an information page or guest view, even if the course or group room is unavailable.  Tenants, as a group room, also have an information page.  The personal office page can be essentially the profile of the person and special other rooms will have to create an information page, if they have them at all. (3) Clicking the owner of the room brings up that member’s profile.

Rooms may be unavailable if they are locked or are course and group rooms for which the user is not a member.  Private courses and groups do not appear in the Places tab at all (or the directory tabs at all) unless the user is a member of them.  Courses rooms, group rooms, and personal offices are also accessible by way of the Courses, Groups, and People sub-tabs, respectively.  No owner is listed for meta-group rooms.  Like the People tab, if there are too many results, the room list will have to be broken into chunks.  Some options include by category, by tenant, or into alphabetical chunks by room name.
