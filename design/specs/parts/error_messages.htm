error messages

·      Two types of errors:

o      Something changed by TI system (i.e. removal or change to a TI-specific room or group) or another user (i.e. deletes a file, group or course), user is notified via PM message. In the case of a group room being deleted while user is in it, they are transferred to their own office and receive an automatic PM message.

o      If user does something, the error message

§       appears at the top of the page (another page does not open with the error message on it)

§       is in a distinctive color that stands out against the background

·      Error messages are short and informative

·      Error messages should be necessary.

o      If a button can be grayed or an option not presented so a user does not receive an error message, this should be the preference over having an error message.

o      In some cases, more than a gray button will be necessary. For example, when the number of whiteboards (3) has been reached text near the grayed button should explain “Maximum of 3 simultaneous whiteboards”.

·      All deletes are confirmed with a statement such as “All configurations for this Group/Passageway/Course will be lost when it is deleted. Do you want to delete it and its configurations?”   |Yes|       |Cancel|

·      For areas that select users, rooms, groups, and courses for preferences (Favorites, Buddies, Passageways, etc.), searching for that same room, groups, and courses will display the room, groups, and courses as grayed since it cannot be selected again under the same preference.
