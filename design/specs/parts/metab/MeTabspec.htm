1.   Overview

The Me tab directory is a pervasive tab in the Tapped In environment ; the general characteristics of which are described in “Tapped In” and “Tenant” Directories Functional Specification.

 

The Me tab can be thought of as the users backpack, items that she or he can take with them as they change locations and interact with others in Tapped In.

 

 

2.   Sub Tabs

The subtabs under the Me tab are:

 

  • Welcome: An overview of what can be done in the Me Tab.
  • People: A list of people the user deems as favorites
  • Places: A list of and access to rooms for groups, courses, offices, to which the user is a member and TI public rooms. This list is automatically generated when the user signs up for any of the above. User can also add to this list.
  • Courses: A list of and access to courses in which the user is enrolled.
  • Groups: A list of and access to groups in which the user is currently a member.
  • Files: User can upload files to this area to be shared from other parts of TI. (See Files Spec for details.). Files can be shared with others.
  • Links: User can store links here for easy access. (See Links Spec for details). Links can be shared with others.
  • Calendar: personal calendar to which all courses and group events are listed (See Calendar Spec for details.)
  • Settings: User can set their preferences

2.3  Places 2

From Places, a user can add and delete, go to, categorize, and mark for shortcuts their favorite places.

 

Adding and Deleting Places

  • Adding a place to the favorites list works similar to Passageways where user does a search that produces a list of possible matches. User checks the match, and then adds it to the list of favorites (one difference is Categorization of Places; see Categorization below)
  • Any group or course room to which the user belongs automatically appears in Favorite Places; the user can remove it.
  • Users can create their My office from Favorite Places; if they have an office, it automatically is listed as a favorite and appears in the shortcuts pull down menu.
  • Deleting a place follows the same pattern as removing one’s self from a group or course.
    • When the delete link is clicked, a message appears at the top of the page explaining that the place will be deleted from Favorite Places.
    • The message ends with asking them to |Confirm|  or |Cancel| their request.
    • Once a member deletes a place, a message appears at the top of the page stating that the place has been deleted from their favorites and the information no longer appears on the page.

 

Linking to your Favorite Places and Shortcuts

  • All names of places link to the Welcome page or Guestview page depending on the situation.
  • Users can mark a place to appear in their shortcut menu; the button marked Favorite Places on the main screen of Tapped In.
  • A user’s My Office automatically appears in the shortcut menu if they have an office.
  •  

Categorization

In order to organize the lists of Favorite Places, there are four categories that a user can put a Favorite Place: Community-wide; Courses; Groups; and Offices.

  • Community-wide places have a set default list since these are TI community rooms, open to all. A user may remove these rooms.
  • Community wide places are on the first page of places; the rest of categories can be navigated to.
  • My Office is automatically categorized as an Office.
  • When places are added, a user may select a category. The choice does default to a likely choice based on the TI’s systems categories.
  • The user can change the categorization latter if they wish.

 

 

2.4  Courses

The Couse page allows the user to edit the following:

  • How they receive communication from the Course (Communication includes synchronous and asynchronous). 2

·      They can set this communication by course

·      Or all communication across courses can be the same

  • Their current status in the course: Student; Instructor; Teaching Assistant 2
  • The ability to remove themselves from any of the courses they are enrolled in 1
    • When the remove link is clicked, a message appears at the top of the page explaining that they will be removed from this course, effectively deleting them from the class list
    • The message ends with asking them to |Confirm|  or |Cancel| their request.
    • Once a member removes themselves from a course, a message appears at the top of the page stating that they are no longer enrolled and the course information no longer appears on the page.

The user also has a direct link to the Welcome page of each of their course from this page. 1

If the user is an instructor or teaching assistant, there is a direct link to the Settings for the course as well. 1

If the user is a student in a class that belongs to a project group, the group is listed under the course and linked to the group’s Main/Welcome page. 2
