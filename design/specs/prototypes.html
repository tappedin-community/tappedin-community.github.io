Overview

Interface design and functional specifications were completed July 2001 - Feb 2002, and included about 20 design interations (see mockup evolution) and an interactive demo.

Development planning (evaluation of architectural frameworks, selection, and familiarization; database design; development environment setup; cvs setup, etc.) took most of February and March 2002. Implementation began late March 2002. Within each task specification, we focused on completing priority 1; features (indicated with [1] in the specifications) for the Alpha release.

We released our Alpha candidate on Sept 3, 2002. We expect to release a Beta candidate in October, and a Public release on June 1, 2003.

Development Notes

After a review of several candidate architectures, the TI2 technology team (Larry, Zaz, Patti) decided to build on JCorporate's Expresso Framework. Expresso is a three-tier, model-view-controller architecture implemented in Java, build on top of the popular Jakarta Struts Framework. We are also applying this framework to another SRI project called PADI also currently under development by the same team of developers.

Our chat server and client implementation is based on NFCChat and OpenJMS.

All of these packages (Expresso, NFCChat, OpenJMS) are 100% Java, open source, and have licenses that allow commercial sale of derivative products. Our development system is run daily on Windows, Mac OS X, and Solaris; it is a demonstrably cross-platform server solution, which is good new for clients who may want to run their own instance of TAPPED IN on their own servers.

More about Expresso, from the Jcorporate Website:

Expresso is a... "Web Application Development (AD) Framework designed to shorten time-to-delivery of Web-based, business transactional applications. An open standards based library of extensible Java Server components for providing services to web-based applications so the developer can concentrate on application logic. Expresso is a powerful and broad architectural framework with extensible Java components for developing database-driven web applications based on open standards. Expresso integrates with Struts which is a MVC light weight framework emphasizing presentation and application configuration, and bringing a powerful tag library to Expresso; whereas Expresso adds capabilities for database-stored security, robust object-relational mapping, background job handling and scheduling, self-tests, log4j logging integration, automated table manipulation, database connection pooling, email connectivity, event notification, caching, internationalization, XML automation, testing, registration objects, configuration management, automatic database maintenance and JSP tag library etc. Use Expresso in both non EJB/J2EE sites, as well as scale upwards using J2EE™ technology. Expresso enables developers to decrease their learning curve and time to solution, which translates directly into reduced costs and increased revenue for customers."

In a nutshell, the Exresso Framework is a large toolkit of classes that helps developers create an extensible development environment for their application. The framework can be divided technically into four essential parts: schemas, database object persistence, controller design, and the rest of the kit, which involves job control, utilities, etc.
